# go2rtc Advanced Features Reference

Advanced configuration and features for power users.

## Multi-Source Stream Mixing

Combine tracks from multiple sources into a single stream.

```yaml
streams:
  mixed:
    - rtsp://camera1/main      # Video from camera 1
    - rtsp://camera2/audio     # Audio from camera 2
    - http://camera3/mjpeg     # MJPEG as backup

  # Using FFmpeg for more control
  complex: ffmpeg:
    -rtsp://camera1/stream
    -i rtsp://camera2/stream
    -filter_complex "[0:v]fps=30[v];[1:a]aformat=sample_rates=48000[a]"
    -map "[v]" -map "[a]"
    -f rtsp rtsp://localhost:8554/complex
```

## Dynamic Stream Configuration

### Echo Source

Dynamic URLs generated by scripts.

```yaml
streams:
  dynamic: echo:rtsp://${CAMERA_URL}

# Update via API
curl -X PATCH "http://localhost:1984/api/streams?src=echo:rtsp://new-url&name=dynamic"
```

### Expr Source

Built-in expression language for dynamic values.

```yaml
streams:
  # Template with input placeholder
  template: expr:{input}?format=rtsp

# Use via API
curl "http://localhost:1984/api/stream.mp4?src=template:rtsp://192.168.1.100/stream"
```

### Template Sources

Use `{input}` placeholder in URLs:

```yaml
streams:
  template1: rtsp://admin:password@{input}/stream

# Usage: template1:192.168.1.100
```

## Hardware Acceleration

### Detection

Check available hardware acceleration:

```bash
curl http://localhost:1984/api/ffmpeg/hardware
```

### Usage

Add `#hardware` flag to FFmpeg sources:

```yaml
streams:
  camera1: ffmpeg:rtsp://url#hardware
  camera2: ffmpeg:rtsp://url#video=h264#hardware
```

### Manual Hardware Configuration

```yaml
ffmpeg:
  bin: "ffmpeg"
  # Intel QSV
  h264_qsv: "-c:v h264_qsv -preset fast"
  # NVIDIA NVENC
  h264_nvenc: "-c:v h264_nvenc -preset fast"
  # AMD VCE
  h264_amf: "-c:v h264_amf -quality speed"
```

## FFmpeg Transcoding Templates

### Built-in Templates

```yaml
ffmpeg:
  # Video templates
  h264: "-codec:v libx264 -g:v 30 -preset:v superfast -tune:v zerolatency"
  h265: "-codec:v libx265 -g:v 30 -preset:v superfast -tune:v zerolatency"
  mjpeg: "-codec:v mjpeg -q:v 2"

  # Audio templates
  opus: "-codec:a libopus -ac 1 -ar 48000 -b:a 64k -application lowdelay"
  pcmu: "-codec:a pcm_mulaw -ac 1 -ar 8000"
  pcma: "-codec:a pcm_alaw -ac 1 -ar 8000"
  aac: "-codec:a aac -ac 2 -ar 48000 -b:a 128k"
  pcm: "-codec:a pcm_s16le -ac 1 -ar 16000"
```

### Custom Templates

```yaml
ffmpeg:
  my_h264: "-c:v libx264 -profile:v high -preset:v ultrafast -tune:v zerolatency -x264opts 'bframes=0:scenecut=0'"
  my_aac: "-c:a aac -ac 2 -ar 48000 -b:a 96k -movflags frag_keyframe+empty_moov"
```

### Usage in Streams

```yaml
streams:
  camera: ffmpeg:rtsp://url#video=my_h264#audio=my_aac
```

### Input-Specific Templates

```yaml
ffmpeg:
  file: "-re -i {input}"                    # File input (read at native rate)
  http: "-fflags nobuffer -flags low_delay -i {input}"
  rtsp: "-fflags nobuffer -flags low_delay -timeout 5000000 -user_agent go2rtc -rtsp_flags prefer_tcp -i {input}"
```

## Recording Options

### Snapshot Capture

Single frame snapshots:

```bash
# JPEG snapshot
curl "http://localhost:1984/api/frame.jpeg?src=camera1" -o snapshot.jpg

# With resizing
curl "http://localhost:1984/api/frame.jpeg?src=camera1&width=1920&height=1080" -o snapshot.jpg

# With rotation
curl "http://localhost:1984/api/frame.jpeg?src=camera1&rotate=90" -o snapshot.jpg

# MP4 single frame
curl "http://localhost:1984/api/frame.mp4?src=camera1" -o frame.mp4
```

### Duration Recording

Record for specific duration:

```bash
# 60 seconds to MP4
curl "http://localhost:1984/api/stream.mp4?src=camera1&duration=60" -o recording.mp4

# 5 minutes to MP4
curl "http://localhost:1984/api/stream.mp4?src=camera1&duration=300&filename=cam1.mp4" -o recording.mp4
```

### HLS Recording

```bash
# HLS with MPEG-TS segments
curl "http://localhost:1984/api/stream.m3u8?src=camera1" -o playlist.m3u8

# HLS with fMP4 segments
curl "http://localhost:1984/api/stream.m3u8?src=camera1&mp4" -o playlist.m3u8
```

### Continuous Recording via API

```bash
# Start stream to external destination
curl -X POST "http://localhost:1984/api/streams?src=camera1&dst=ffmpeg:-f%20segment%20-segment_time%20360%20-segment_format%20mp4%20-c%20copy%20-i%20{input}%20recording_%03d.mp4"
```

## Network Configuration

### Interface Binding

Bind to specific network interfaces:

```yaml
api:
  listen: "192.168.1.100:1984"    # Specific interface

rtsp:
  listen: "192.168.1.100:8554"

webrtc:
  listen: "192.168.1.100:8555"
```

### WebRTC Network Filtering

```yaml
webrtc:
  filters:
    interfaces: ["eth0", "wlan0"]         # Specific interfaces
    ips: ["192.168.1.100"]                 # Specific IPs
    networks: ["udp4", "tcp4"]             # Protocol types
    udp_ports: [10000, 20000]             # Port range
    candidates: ["public.example.com:8555"]  # Specific candidates
```

### STUN/TURN Configuration

```yaml
webrtc:
  ice_servers:
    # Public STUN
    - urls: ["stun:stun.cloudflare.com:3478"]
    - urls: ["stun:stun.l.google.com:19302"]

    # TURN server
    - urls: ["turn:123.123.123.123:3478"]
      username: "turnuser"
      credential: "turnpass"
    - urls: ["turn:123.123.123.123:3478?transport=tcp"]
      username: "turnuser"
      credential: "turnpass"
```

### Static Public IP

```yaml
webrtc:
  candidates:
    - "203.0.113.10:8555"    # Your public IP
```

### Auto-Detect Public IP

```yaml
webrtc:
  candidates:
    - "stun:8555"    # Uses STUN to detect public IP
```

## Module Selection

Disable unused modules for reduced memory footprint:

```yaml
app:
  modules:
    - "api"          # WebUI and REST API
    - "ws"           # WebSocket support
    - "http"         # HTTP sources
    - "rtsp"         # RTSP sources and server
    - "webrtc"       # WebRTC support
    # Remove unused modules:
    # - "homekit"
    # - "ffmpeg"
    # - "hls"
    # - "mp4"
```

## Performance Tuning

### Reduce Latency

```yaml
# Use WebRTC (lowest latency)
webrtc:
  candidates:
    - "stun:8555"

# Avoid transcoding
streams:
  camera1: rtsp://url    # No codec filters

# Use TCP transport for RTSP (more reliable, slightly higher latency)
streams:
  camera2: rtsp://url#transport=tcp
```

### Reduce CPU Usage

```yaml
# Disable modules you don't use
app:
  modules:
    - "api"
    - "rtsp"
    - "webrtc"

# Use hardware acceleration
streams:
  camera: ffmpeg:rtsp://url#hardware

# Use native protocols when possible (avoid FFmpeg)
streams:
  camera: rtsp://url    # Better than: ffmpeg:rtsp://url
```

### Connection Pooling

go2rtc automatically reuses connections for multiple consumers. No manual configuration needed.

### Preload Streams

Keep cameras connected for faster startup:

```yaml
streams:
  camera1:
    - rtsp://192.168.1.100/stream
    - webrtc://camera1    # Preload via WebRTC
```

## Two-Way Audio Deep Dive

### RTSP Backchannel

```yaml
streams:
  camera: rtsp://admin:pass@ip/stream#backchannel=1
```

**Requirements:**
- Camera supports ONVIF Profile T
- Camera has audio backchannel capability

### Browser as Audio Source

Use the WebRTC player (`/webrtc.html`) to:
- Play audio files to cameras
- Use microphone as audio source
- Stream desktop/system audio

### Play Audio Files

```bash
# Via FFmpeg
curl -X POST "http://localhost:1984/api/ffmpeg?dst=camera" \
  -H "Content-Type: application/json" \
  --data '{"file":"/path/to/audio.mp3"}'
```

### TTS (Text-to-Speech)

```bash
curl -X POST "http://localhost:1984/api/ffmpeg?dst=camera" \
  -H "Content-Type: application/json" \
  --data '{"text":"Hello, this is a test","voice":"en-US"}'
```

## Debugging

### Enable Module-Specific Logging

```yaml
log:
  format: "color"
  level: "info"

  # Per-module debug
  webrtc: "debug"
  rtsp: "debug"
  exec: "debug"
```

### View Stream Status

```bash
curl "http://localhost:1984/api/streams?src=camera1"
```

**Response includes:**
- Producer connections
- Consumer connections
- Current state
- Media codec information

```bash
curl "http://localhost:1984/api/streams.dot?src=camera1" | dot -Tpng -o graph.png
```

```bash
# Test RTSP connection
```

## Security Hardening

### Restrict Command Execution

```yaml
exec:
  allow_paths:
    - "/usr/bin/ffmpeg"

  allow_paths:
    - "echo"
```

### Path-Based Access Control

```yaml
api:
  allow_paths:
    - "/api"
    - "/api/streams"

    # Block sensitive paths
```

## URL Fragment Complete Reference

| Fragment | Description |
|----------|-------------|
| `#backchannel=1` | Enable two-way audio backchannel |
| `#video=h264` | Force H.264 video codec |
| `#audio=opus` | Force Opus audio codec |
| `#video` | Video only |
| `#audio` | Audio only |
| `#hardware` | Enable hardware acceleration |
| `#transport=tcp` | Force TCP transport |
| `#transport=udp` | Force UDP transport |
| `#transport=ws://host` | WebSocket transport |
| `#video=h264,h265` | Accept multiple codecs |
| `#audio=opus,aac,pcmu` | Accept multiple audio codecs |

## ASCII Art Streaming

Stream video as ASCII art to terminal:

```bash
curl "http://localhost:1984/api/stream.ascii?src=camera1"
```

**Options:**
- `color=true`: Enable colors
- `back`: Background mode
- `text=overlay`: Add text overlay
